Business Automations Card - Static Image Fix

Problem Identified:

The Business Automations card was showing a loading screen for a video instead of properly displaying the static image.

Root Cause:

Even though the card was configured with isStaticImage: true, it was still using the useVideoLazyLoad hook which initializes with isLoading: true and shows a loading spinner.

Before (Issue):

  • ❌ Loading spinner displayed
  • ❌ Video-related hooks executed
  • ❌ Image appeared after delay
  • ❌ Unnecessary video processing

After (Fixed):

  • ✅ No loading spinner
  • ✅ Bypassed video loading logic
  • ✅ Image loads immediately
  • ✅ Optimized for static images

Key Changes Made:

// Modified VideoCard component to handle static images properly const videoHook = isStaticImage ? null : useVideoLazyLoad({...}); // Set appropriate states for static images const isInView = isStaticImage ? true : videoHook?.isInView || false; const isLoaded = isStaticImage ? true : videoHook?.isLoaded || false; const isLoading = isStaticImage ? false : videoHook?.isLoading || false; // Skip video-related useEffect hooks for static images useEffect(() => { if (isStaticImage) return; // ... video logic }, [isStaticImage]); // Optimized image rendering for static images - Set isStaticImage to true to bypass video loading logic - Image loads immediately with loading="eager" and priority={true} - No loading spinner or video controls are rendered - Text overlay only appears if title or description exists // Key implementation changes: 1. Conditionally use useVideoLazyLoad hook only for videos 2. Set isLoading to false for static images 3. Set isLoaded to true for static images 4. Skip all video-related useEffect hooks for static images 5. Use optimized Image component with eager loading

Test Result:

✅ The Business Automations card now displays correctly as a static image without any loading states or video-related functionality.

  5. Verify no loading spinner appears
  6. Check that no video controls are visible
  7. Confirm no text overlay appears (since title and description are empty)
